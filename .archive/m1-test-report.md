## 部署问题与解决

### 问题 1: Node.js 路径
- **现象**: systemd 服务启动失败
- **原因**: ExecStart 使用 `/usr/bin/node`，实际路径是 `/usr/local/bin/node`
- **解决**: 修改 systemd 配置使用正确路径

### 问题 2: 腾讯云安全组
- **现象**: 外部无法连接 8080 端口
- **原因**: 云安全组未开放端口
- **解决**: 翎麟手动配置云安全组

## 关键发现

1. **多层防火墙**: 云服务器有本地 iptables + 云安全组两层防火墙
2. **分层诊断有效**: 本地测试 → ping → 端口扫描 → iptables → 云安全组
3. **systemd 路径**: 需要使用完整路径，不能依赖 PATH 环境变量
